### Διεπαφή Προγραμματισμού Εφαρμογών - API

Η διεπαφή προγραμματισμού εφαρμογών (**A**pplication **P**rogramming **I**nterface) είναι μία διεπαφή των προγραμματιστικών διαδικασιών που παρέχει ένα λογισμικό, ένα λειτουργικό σύστημα ή μία βιβλιοθήκη προκειμένου να επιτρέπει την αποστολή αιτημάτων ή την ανταλλαγή δεδομένων με άλλα λογισμικά. Όταν ένα API χρησιμοποιείται για την ανάπτυξη διαδικτυακών εφαρμογών αναφέρεται ώς web API. Ένα web API περιέχει ένα σύνολο από μεθόδους που εκτελούν HTTP requests σε έναν server. Τα web API ανταλλάσσουν δεδομένα σε μορφή JSON ή σε XML . Αν και ο όρος web API ιστορικά ήταν συνώνυμος με τον όρο υπηρεσίες web η σύγχρονη τάση απομακρύνεται από το Simple Object Access Protocol (SOAP) που βασίζεται σε υπηρεσίες web και service-oriented architecture (SOA) και κατευθύνεται σε στυλ representation state transfer (REST), ηλεκτρονικούς πόρους και resource-oriented architecture (ROA). 

### Αρχιτεκτονική Rpresentational State Transfer - REST

Η αρχιτεκτονική REST (**RΕ**presentational **S**tate **T**ransfer) αποτελεί ένα σύνολο από αρχές σχεδίασης 
μιας δικτυακής υπηρεσίας. Η μεταβολή της κατάστασης της βάσης δεδομένων του συστήματος 
περιγράφεται και μεταφέρεται στο σύστημα μέσω του πρωτοκόλλου **HTTP** από διάφορους clients 
ανεξαρτήτως της γλώσσας στην οποία έχουν υλοποιηθεί. Οι βασικές αρχές σχεδίασης του REST είναι οι εξής: 

#### Αποκλειστική χρήση HTTP requests

Αποκλειστική χρήση HTTP αιτημάτων για την επικοινωνία του client app με τον διακομιστή server. Η βασική αρχή σχεδίασης του REST είναι η αντιστοίχηση των 
λειτουργιών CRUD (create, read, update και delete) και HTTP μεθόδων. 
Για την δημιουργία μίας νέας εγγραφής στην βάση δεδομένων γίνεται χρήση της μεθόδου `POST` , για την 
ανάγνωση εγγραφών γίνεται χρήση της `GET` , για την τροποποίηση χρησιμοποιείται η μέθοδος `PUT` και για την διαγραφή μίας εγγραφής από την βάση δεδομένων 
χρησιμοποιείται η μέθοδος `DELETE`. 

#### Είναι stateless

Έλλειψη κατάστασης(stateless) , οποιαδήποτε κλήση σε μια υπηρεσία REST δε θα πρέπει να αναφέρεται σε άλλη προγενέστερη ή μεταγενέστερη κλήση. Ο server δε 
πρέπει να γνωρίζει τι έγινε σε κάποια προηγούμενη κλήση τη στιγμή που επεξεργάζεται την τρέχουσα κλήση. 

#### Απεικόνιση της δομής των καταλόγων σαν URIs.

Ο σχηματισμός των URI ενός REST API πρέπει να είναι εύκολος και προβλέψιμος από τον χρήστη της υπηρεσίας και να απαιτείται από λίγη έως καθόλου τεκμηρίωση σχετικά με το resource το οποίο αντιπροσωπέυει κάθε URI. 
Ένας τρόπος για να γίνει αυτό είναι να ορίσουμε URI παρόμοια με τον τρόπο δόμησης των καταλόγων σε ένα σύστημα αρχείων. 
Για παράδειγμα η δομή ενός URI που δείχνει σε ένα συγκεκριμένο βιβλίο μέσα σε ένα σύνολο βιβλίων θα μπορούσε να είναι:

```
GET /books/{id}
```

μία πιο σύνθετη αναζήτηση θα μπορούσε να είναι

```
GET /books/{year}/{day}/{month}
```

#### Μεταφορά δεδομένων μέσω XML και JSON 

Για να δώσουμε στον client την δυνατότητα να επιλέξει τη μορφή των περιεχομένων της απόκρισης του server στην πιο κατάλληλη γι αυτόν,δεν έχουμε παρά να φτιάξουμε την υπηρεσία μας έτσι ώστε να κάνει χρήση του ενσωματομένου HTTP Accept header όπου η τιμή του θα είναι ένας τύπος MIME. Οι πιο συνηθισμένοι τύποι μορφοποίησης δεδομένων στις REST υπηρεσίες είναι JSON - `application/json`, XML - `application/xml` και XHTML - `application/xhtml+xml`.
